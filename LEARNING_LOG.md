## 2026-07-18 第1〜7章

### 詰まった点と解決策

| 詰まった点 | 原因 | 解決策 |
|---|---|---|
| OpenAI APIが`429`エラー | 無料トライアル残高が$0 | クレジットをチャージ |
| Googleログインで`redirect_uri_mismatch` | Google Cloud Console側の設定反映待ち・複数クライアントの混同 | 正しいクライアントIDのリダイレクトURIを再確認、数分待つ |
| 画面が真っ白で何も表示されない | `let supabase`がCDNライブラリの`supabase`と名前衝突 → 構文エラーでスクリプト全体が停止 | 変数名を`supabaseClient`に変更 |
| `usage`テーブルへの書き込みが効かない | RLSポリシーはあってもGRANT(基本アクセス権限)が無く`permission denied` | `service_role`への`grant`文を追加 |

### 3つの学び

1. **エラーが起きたら、何よりも先にブラウザの「コンソール」を見る**。画面が真っ白でも、コンソールにはほぼ答えが書いてある。
2. **RLSとGRANTは別物**。「行ごとに絞り込むルール(RLS)」と「そもそもテーブルに触れる権限(GRANT)」は両方揃って初めて機能する。
3. **強い権限(service_role key)はサーバーだけに置く**。ユーザー個人のデータはRLS+anon keyで守り、集計・カウンターのような「ユーザーに操作させたくないもの」だけservice_roleを使う。
