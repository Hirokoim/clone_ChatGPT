【第5章でのつまずき報告】外部ライブラリの読み込み方によって、コードが「無言で」全滅することがある

何が起きたか

第5章(会話をDBに保存する)を実装する際、SupabaseというサービスをブラウザのJavaScriptから使うために、「CDN」と呼ばれる方法(外部のURLから直接ライブラリを読み込む方法)を使いました。

<script src="https://unpkg.com/@supabase/supabase-js@2"></script>
この読み込み方をすると、Supabaseのライブラリは自動的にsupabaseという名前の「置き場所」をブラウザの中に作ります。

その後、自分のコードの中でも「Supabaseを使う準備をした結果を入れる箱」として、同じくsupabaseという名前を使ってしまいました。すると、同じ名前の箱を二重に作ろうとしたことになり、ブラウザはエラーを起こしてスクリプト全体を実行しなくなりました。

何が厄介だったか(非エンジニアに伝えたいポイント)

エラーが起きても画面には何も表示されない。 ボタンも文字も何も出ず、真っ白な画面になるだけなので、「何かが動いていない」ことは分かっても「何が原因か」は画面を見ただけでは全く分からない
原因はブラウザの「開発者ツール」の中の「コンソール」というところにしか表示されない。 普段は開かない場所なので、そこを見る発想自体が非エンジニアには難しい
逆に言うと、コンソールさえ見れば、エラーメッセージにほぼ答えがそのまま書いてある(Identifier 'supabase' has already been declared =「supabaseという名前は既に使われています」)
教材作成者への提案

第5章でSupabaseをブラウザから使う設定をする際、もし「CDNで読み込む」方法を教材に含める場合は、

「ライブラリが自動で使う名前(supabase)と、自分のコードの変数名は絶対に被らせないこと」という一言の注意書き
エラーが起きたらまずブラウザの開発者ツールのコンソールを見るという手順を、早い段階(第1〜3章あたり)で一度教えておく
の2点があると、同じ場所でつまずく人がかなり減ると思います。
